Chest — RPG Loot & Storage System

Blood Moon InteractiveRPG Systems

The Chest behavior is a powerful storage system in GameGuru MAX that brings classic RPG inventory management to your games. Whether you're creating treasure chests, supply caches, or merchant containers, this behavior handles all the complexity of persistent storage and HUD integration.

Key Feature

Persistent storage containers with HUD screen integration and customizable interaction ranges make it easy to create authentic RPG experiences where players can collect, organize, and manage their inventory through intuitive container interactions.

Video Tutorial

Why Chests Matter in RPG Design

Chests and storage containers are fundamental to RPG gameplay. They serve multiple purposes:

  • Inventory Management: Provide safe storage for players to organize their items
  • Loot Rewards: Create satisfying moments when players discover treasure
  • World Building: Chests tell stories about dungeons, merchants, and civilizations
  • Game Progression: Control what items are available at specific story points
  • Player Agency: Let players decide what to carry and what to store

Setup & Configuration

  1. Create a 3D object in your scene to represent the chest (model or placeholder).
  2. Set the object's AlwaysActive property to ON.
  3. Attach the chest behavior from the RPG behaviors category.
  4. Configure the USE_RANGE to control how far players can interact with the chest.
  5. Set USE_PROMPT to the text players will see ("Press E to open", etc.).
  6. Choose or create a HUD screen for the container interface with CHEST_SCREEN.
  7. Name your container uniquely with CHEST_CONTAINER (use "chestunique" for per-chest storage).
  8. Configure prompt display location and visual highlighting as desired.
  9. Add sound effects to Sound0 and Sound1 for opening/closing audio feedback.

Core Properties

Interaction Settings

  • USE_RANGE (50-200 units, default 100): How close the player must be to interact. Longer ranges make chests more discoverable, shorter ranges encourage exploration.
  • USE_PROMPT: The text displayed when a player can interact ("Press E to open", "Loot this chest", etc.)
  • PROMPT_DISPLAY: Choose between Local (near the chest) or Screen (center of HUD)

Storage Configuration

  • CHEST_SCREEN: The HUD screen that opens when the chest is accessed. Create custom HUD screens for different container types.
  • CHEST_CONTAINER: The internal name for this chest's inventory. Use unique names to give each chest its own storage, or use shared names for linked containers.

Visual Enhancement

  • ITEM_HIGHLIGHT: Visual feedback options — None, Shape (highlight the object), or Outline (glow effect) — help players spot interactive chests.

Practical Implementation Examples

Treasure Chest in a Dungeon

Create an iconic dungeon treasure chest with:

  • USE_RANGE: 80 (players must approach closely)
  • USE_PROMPT: "E to loot treasure"
  • ITEM_HIGHLIGHT: Shape (make it obviously interactive)
  • CHEST_CONTAINER: "treasureChest_Level3" (unique per level)
  • Sound0: Wood creak effect for opening

Merchant Supply Crate

Set up a merchant's storage area:

  • USE_RANGE: 120 (more accessible)
  • USE_PROMPT: "Browse supplies"
  • CHEST_SCREEN: Custom merchant HUD
  • CHEST_CONTAINER: "merchantSupplies" (shared inventory across all crates)
  • PROMPT_DISPLAY: Screen (better for shop experience)

Player Home Storage

Multiple chests in a player's base or home:

  • USE_RANGE: 100
  • USE_PROMPT: "Organize items"
  • Create separate containers for each room: "homeChest_Bedroom", "homeChest_Storage", etc.
  • Allows players to organize their inventory across multiple locations

Advanced Techniques

Shared vs. Unique Storage

The CHEST_CONTAINER name determines whether chests share inventory:

  • Unique Names: Each chest has its own storage ("chest_01", "chest_02")
  • Shared Names: Multiple chests link to the same inventory ("armory" links all weapon caches)

Conditional Chest Contents

Populate chests dynamically based on story progression:

  • Use logic connections to activate/deactivate items based on player level
  • Create boss-specific loot chests that only contain rewards after defeating enemies
  • Reference global variables to determine what items should be available

Chest Highlighting for Discovery

Make hidden chests more discoverable with visual feedback:

  • Use the Shape or Outline highlight option to gently guide players
  • Combine with proximity-based lighting effects for atmospheric discovery

Common Pitfalls & Solutions

Problem Solution
Chest won't open Verify the HUD screen name exists and is created in your project
Items disappear after collection Ensure each chest has a unique CHEST_CONTAINER name if you want persistent storage
Player can't interact from far away Increase USE_RANGE value (try 150-200 for more accessible chests)
Multiple chests share same contents This is intended if using the same CHEST_CONTAINER name — change to unique names for independent storage
Prompt not visible Change PROMPT_DISPLAY to "Screen" for better visibility, or increase ITEM_HIGHLIGHT

Design Best Practices

Feedback and Polish

  • Always add sound effects: Creaking wood, metal latches, or magical chimes make interactions feel substantial
  • Visual highlighting: Even subtle outlines help players recognize interactive objects
  • Clear prompts: Write prompts that match your game's tone ("Loot", "Open", "Search")

Inventory Flow

  • Keep USE_RANGE reasonable — too far feels unrealistic, too close feels restrictive
  • Create multiple small chests rather than one huge container for better pacing
  • Use consistent HUD screens across similar chest types for familiarity

Narrative Integration

  • Place chests intentionally — they should reward exploration and curiosity
  • Consider what a chest contains and why — treasure feels more rewarding when it's appropriate to the location
  • Use chests to tell environmental stories (a ransacked chest suggests previous adventurers, etc.)

Related Behaviors

The Chest behavior works excellently with these other systems:

  • Inventory System: Container integration for item management
  • HUD System: Screen display and interface management
  • Loot System: Populate chests with random or predetermined rewards
  • Global Variables: Track which chests have been opened
  • Object Interaction: Combine with other interactive objects for complex scenarios
