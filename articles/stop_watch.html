Stop Watch — Timed Challenges & Displays

Blood Moon InteractiveHUDs

The Stop Watch behavior provides a flexible timer system for GameGuru MAX — ideal for countdowns, speedruns, timed puzzles, and any mechanic that requires precise time tracking and configurable end actions.

Key Feature

Multiple readout formats, launch warnings, global variable display, and configurable end actions make this behavior ideal for a wide range of gameplay scenarios.

Video Tutorial

Setup

  1. Create an object to represent the stopwatch.
  2. Set `AlwaysActive` to ON if you want it to run without external activation.
  3. Attach the `stop_watch` behavior (`objects/stop_watch.lua`).
  4. Configure `DISPLAY_USER_GLOBAL` with a global variable name to display the time.
  5. Choose the desired `READOUT` format and `END_ACTION`.

Properties

  • DISPLAY_USER_GLOBAL: Global variable name to display the time.
  • END_ACTION: What happens when the timer stops (None, Hurt Player, Win Game, etc.).
  • LAUNCH_WARNING: Optional countdown before the timer begins.
  • END_SCREEN: HUD screen to display when END_ACTION is `Display Hud Screen`.
  • READOUT: Time format (e.g., Mins:Secs:MilliSecs).

Examples

Speedrun Timer

Attach the behavior to a central HUD object and set `DISPLAY_USER_GLOBAL` to `RunTime`. Use `READOUT` set to include milliseconds for precise timing. Configure end actions to record or announce completion.

Countdown Puzzle

Use `LAUNCH_WARNING` to show a 3-2-1 countdown. Set `END_ACTION` to `Lose Game` or `Activate Entities` to trigger traps or doors when time runs out.

Tips & Troubleshooting

  • If your global variable isn't updating, verify the variable name matches exactly and that the HUD reads that global.
  • For multiplayer or replicated systems, ensure global handling is compatible with your networking setup.
  • Use sounds in `Sound0`/`Sound1` for start/stop feedback.
